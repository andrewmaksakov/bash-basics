# RAID

## Что это такое

**RAID (Redundant Array of Independent Disks)** — технология объединения нескольких физических дисков в один логический массив для повышения скорости и/или надёжности.

## Для чего нужно

- Один диск может **сломаться** → данные потеряны. RAID решает: данные дублируются на другие диски.
- Один диск **медленный** → работа тормозит. RAID решает: чтение/запись параллельно на несколько дисков.

---

## Ключевые понятия

| Термин | Что это |
|--------|--------|
| **Stripe** (страйп) | Режет данные на куски и пишет на разные диски параллельно. Даёт скорость |
| **Mirror** (зеркало) | Пишет одни и те же данные на 2 диска. Даёт надёжность |
| **Parity** (паритет) | Контрольная сумма. Позволяет восстановить данные с упавшего диска |
| **Hot spare** | Запасной диск, который автоматически включается если один умер |

---

## Основные уровни RAID

| RAID | Что делает | Мин. дисков | Скорость | Надёжность | Полезный объём |
|------|-----------|-------------|----------|-----------|----------------|
| **0** | Stripe — режет данные на куски и пишет параллельно на все диски | 2 | Очень быстро | НИКАКОЙ — один диск умер → всё потеряно | 100% |
| **1** | Mirror — пишет одни и те же данные на 2 диска | 2 | Запись медленнее, чтение быстрее | Можно потерять 1 диск | 50% |
| **5** | Stripe + parity — данные + контрольная сумма распределены по дискам | 3 | Быстро | Можно потерять 1 диск | (N-1)/N |
| **6** | Stripe + двойная parity | 4 | Чуть медленнее 5 | Можно потерять 2 диска | (N-2)/N |
| **10** | Комбо 1+0 — зеркало + stripe | 4 | Очень быстро | Можно потерять диски в разных парах | 50% |

---

## Подробно по каждому уровню

### RAID 0 — Stripe
Данные режутся на куски и пишутся параллельно на все диски.

- **Плюс:** очень быстро (несколько дисков работают одновременно)
- **Минус:** ноль надёжности — один диск умер → потерял ВСЁ
- **Где используют:** кеши, видеомонтаж, временные данные

```
Диск1: [A1] [A3] [A5]
Диск2: [A2] [A4] [A6]
```

### RAID 1 — Mirror
Одни и те же данные пишутся на 2 диска одновременно.

- **Плюс:** один диск умер → второй работает, данные целы
- **Минус:** теряешь 50% объёма (2 диска → доступен только 1)
- **Где используют:** системные диски, важные данные

```
Диск1: [A1] [A2] [A3]
Диск2: [A1] [A2] [A3]   ← копия
```

### RAID 5 — Stripe + Parity
Данные + контрольная сумма распределены по всем дискам. Минимум 3 диска.

- **Плюс:** баланс скорости/надёжности/объёма
- **Минус:** если умрут 2 диска одновременно — всё потеряно
- **Где используют:** файловые хранилища, серверы

```
Диск1: [A1] [B1] [Cp]
Диск2: [A2] [Bp] [C1]
Диск3: [Ap] [B2] [C2]
                ↑ parity
```

### RAID 6 — Stripe + двойная Parity
Как RAID 5, но с двумя контрольными суммами. Минимум 4 диска.

- **Плюс:** переживает смерть 2 дисков одновременно
- **Минус:** запись чуть медленнее RAID 5
- **Где используют:** большие хранилища с критичными данными

### RAID 10 — комбо 1+0
Зеркало (RAID 1) + страйп (RAID 0). Минимум 4 диска.

- **Плюс:** очень быстро + надёжно (можно потерять диски в разных парах)
- **Минус:** теряешь 50% объёма
- **Где используют:** базы данных, нагруженные сервисы

```
Зеркало 1:    Зеркало 2:
Диск1 == Диск2    Диск3 == Диск4
       ↓
   Stripe между зеркалами
```

---

## Что значит "parity"

Контрольная сумма. Если один диск умер, по parity можно восстановить его содержимое из оставшихся дисков.

**Пример (упрощённо):**

На трёх дисках лежит `5`, `3`, parity = `5+3=8`.
Диск с `3` умер → берём `8 - 5 = 3` → восстановили.

В реальности используется операция XOR, не сложение, но принцип тот же.

---

## RAID ≠ Бэкап

**Важно:** RAID — это **не бэкап.**

RAID защищает только от **поломки диска**. Не защищает от:
- Удалил файл по ошибке → удалится со всех дисков сразу
- Вирус-шифровальщик → зашифрует данные на всех дисках
- Пожар, кража сервера → массив уничтожен целиком
- Ошибка файловой системы → повредит данные на всех дисках

**Бэкап и RAID — разные задачи. Нужны оба.**

## Шпаргалка

| RAID | Дисков | Что даёт | Объём |
|------|--------|----------|-------|
| 0 | 2+ | Скорость | 100% |
| 1 | 2+ | Надёжность | 50% |
| 5 | 3+ | Скорость + надёжность (теряем 1 диск) | (N-1)/N |
| 6 | 4+ | Скорость + надёжность (теряем 2 диска) | (N-2)/N |
| 10 | 4+ | Максимум скорости и надёжности | 50% |

**Запомни:**
- 0 = stripe, без защиты
- 1 = mirror, копия
- 5 = stripe + 1 parity
- 6 = stripe + 2 parity
- 10 = mirror + stripe
- RAID — НЕ бэкап
