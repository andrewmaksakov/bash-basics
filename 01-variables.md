# 01 — Переменные и аргументы

## Что это такое

**Переменная** — именованная область памяти, в которой хранится значение.

**Аргументы** — значения, которые передаёшь скрипту при запуске.

**Exit code** — число, которое возвращает программа после выполнения (0 = успех, не 0 = ошибка).

**Логирование** — запись событий работы скрипта в файл (что произошло, когда, с каким результатом).

## Для чего нужно

- **Переменные** — чтобы не писать одно и то же значение в коде много раз. Поменял в одном месте — изменилось везде.
- **Аргументы** — чтобы передавать скрипту параметры при запуске (какой файл обработать, сколько строк выводить и т.д.).
- **Exit code** — чтобы проверить успешно ли выполнилась команда, и реагировать на ошибки.
- **Логирование** — чтобы знать что делал скрипт когда тебя не было. Без логов: упал скрипт ночью — непонятно что произошло. С логами: открыл файл — видишь когда упало, на каком шаге и с какой ошибкой.

---

## Переменные — как пользоваться

```bash
NAME="Andrew"        # создали переменную NAME со значением "Andrew"
echo "$NAME"         # достали значение: Andrew
```

**Правила:**
- Между `NAME` и `=` **нет пробелов** — иначе ошибка
- `$NAME` — обращение к значению переменной
- Кавычки `"..."` — позволяют подставлять переменные внутри строки
- Имена пишут заглавными буквами по соглашению

---

## Аргументы — как пользоваться

```bash
bash script.sh file.log 10
#              $1        $2
```

| Переменная | Что это |
|-----------|--------|
| `$0` | Имя самого скрипта |
| `$1` | Первый аргумент |
| `$2` | Второй аргумент |
| `$3`, `$4`... | Третий, четвёртый и т.д. |
| `$#` | Количество аргументов |
| `$@` | Все аргументы как список |

---

## Exit code — как пользоваться

```bash
ls /etc           # команда выполнилась
echo $?           # 0 (успех)

ls /нет_папки     # команда упала
echo $?           # 2 (ошибка)
```

**Завершить скрипт вручную:**
```bash
exit 0    # успешное завершение
exit 1    # завершение с ошибкой
```

## Логирование — как пользоваться

| Оператор | Что делает |
|----------|-----------|
| `>` | Перезаписать файл |
| `>>` | Добавить в конец файла |
| `2>` | Перенаправить только ошибки |
| `&>` | Перенаправить и вывод и ошибки |

```bash
echo "сообщение" > log.txt       # перезаписать
echo "сообщение" >> log.txt      # добавить в конец
echo "$(date): запуск" >> log.txt
```

**`$(команда)`** — выполнить команду и подставить результат.
- `$(date)` — текущая дата и время
- `$(whoami)` — текущий пользователь
- `$(pwd)` — текущая папка

## Шпаргалка

```bash
# Переменные
NAME="Andrew"
echo "$NAME"

# Аргументы
$0    # имя скрипта
$1    # первый аргумент
$#    # количество аргументов
$@    # все аргументы

# Exit code
$?         # код выхода последней команды
exit 0     # успех
exit 1     # ошибка

# Редирект
echo "x" > file       # перезаписать
echo "x" >> file      # добавить в конец
echo "x" 2> err.log   # только ошибки

# Подстановка команды
$(date)
$(whoami)
$(pwd)
```

---

## Полный пример

```bash
#!/bin/bash
# Запуск: bash script.sh hello 42

NAME="Andrew"
echo "Имя: $NAME"
echo "Первый аргумент: $1"
echo "Количество аргументов: $#"

ls /etc > /dev/null
echo "Exit code: $?"

LOG="/tmp/example.log"
echo "$(date): запуск" >> "$LOG"
```
