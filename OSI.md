# OSI

## Что это такое

**OSI (Open Systems Interconnection)** — модель, которая описывает как данные передаются по сети. Делит работу сети на 7 уровней, каждый отвечает за свою задачу.

## Для чего нужно

Чтобы понимать **на каком уровне проблема**, когда что-то не работает.

"Сайт не открывается" — это может быть на 5 разных уровнях. OSI помогает локализовать проблему: на уровне кабеля, IP-адреса, порта, DNS или самого сайта.

---

## 7 уровней OSI (сверху вниз)

| № | Уровень | По-английски | Что делает | Примеры |
|---|---------|--------------|-----------|---------|
| 7 | Прикладной | Application | Программы, которые видит юзер | HTTP, HTTPS, DNS, SSH, FTP |
| 6 | Представления | Presentation | Шифрование, сжатие, кодировка | TLS, JPEG, ASCII |
| 5 | Сеансовый | Session | Установка/закрытие сессии | NetBIOS, RPC |
| 4 | Транспортный | Transport | Доставка пакетов между процессами | TCP, UDP, порты |
| 3 | Сетевой | Network | Маршрутизация между сетями | IP, ICMP, роутеры |
| 2 | Канальный | Data Link | Передача внутри одной сети | MAC-адреса, Ethernet, switch |
| 1 | Физический | Physical | Сами провода и сигналы | Кабели, Wi-Fi, оптоволокно |

---

## Подробно по каждому уровню

### 1. Физический (Physical)
Передача битов (0 и 1) по физической среде. Это сами провода, радиоволны, оптоволокно.

**Примеры:** Ethernet-кабель, Wi-Fi сигнал, оптоволокно.

### 2. Канальный (Data Link)
Передача данных между устройствами **внутри одной сети** (одного сегмента). Использует MAC-адреса.

**Примеры:** Ethernet, switch (свитч), MAC-адрес.

### 3. Сетевой (Network)
Маршрутизация между **разными сетями**. Использует IP-адреса. Решает "как добраться от точки A до точки B через несколько сетей".

**Примеры:** IP, ICMP (ping), роутеры.

### 4. Транспортный (Transport)
Доставка пакетов между конкретными процессами (программами) на компьютерах. Использует порты.

**Примеры:** TCP (надёжно, с подтверждением), UDP (быстро, без подтверждения).

### 5. Сеансовый (Session)
Устанавливает, поддерживает и закрывает сессии (соединения) между приложениями.

**Примеры:** NetBIOS, RPC.

### 6. Представления (Presentation)
Преобразование данных: шифрование, сжатие, кодировка.

**Примеры:** TLS (шифрование HTTPS), JPEG (сжатие картинок), ASCII (кодировка).

### 7. Прикладной (Application)
Протоколы, которыми пользуются программы напрямую.

**Примеры:** HTTP, HTTPS, DNS, SSH, FTP, SMTP.

## Диагностика — на каком уровне проблема

| Симптом | Уровень | Что проверять |
|---------|---------|---------------|
| Нет интернета вообще, нет сигнала | 1 | Кабель, Wi-Fi, питание |
| Сеть видит, но устройства не общаются | 2 | MAC-адреса, switch |
| Не пингуется IP | 3 | IP-адрес, маршрутизация |
| Пингуется, но порт закрыт | 4 | TCP/UDP, firewall, порты |
| Сертификат невалидный | 6 | TLS-сертификат |
| Пингуется по IP, не пингуется по имени | 7 | DNS |
| 404, 500 от сервера | 7 | HTTP, сервер |

## TCP и UDP (4 уровень)

Это два протокола транспортного уровня OSI. Оба доставляют пакеты между процессами, но по-разному.

### Что такое TCP

**TCP (Transmission Control Protocol)** — протокол передачи данных с гарантией доставки. Перед отправкой устанавливает соединение, проверяет что каждый пакет дошёл, при потере — пересылает.

**Как работает:**
1. Устанавливает соединение (3-way handshake: SYN → SYN-ACK → ACK)
2. Режет данные на пакеты, нумерует их
3. Отправляет, получает подтверждение на каждый пакет
4. Если пакет потерян — пересылает
5. Собирает пакеты в правильном порядке на другой стороне
6. Закрывает соединение

**3-way handshake — установка соединения в 3 шага:**

| Шаг | Кто кому | Что значит |
|-----|---------|-----------|
| 1. **SYN** | Клиент → Сервер | "Привет, хочу подключиться" (synchronize) |
| 2. **SYN-ACK** | Сервер → Клиент | "Привет, готов принять" (synchronize + acknowledge) |
| 3. **ACK** | Клиент → Сервер | "Понял, начинаем" (acknowledge) |

После этих 3 шагов соединение установлено, можно передавать данные.

**Расшифровка:**
- **SYN** = synchronize (синхронизация)
- **ACK** = acknowledge (подтверждение)
- **SYN-ACK** = одновременно SYN и ACK в одном пакете

**Зачем 3 шага:** чтобы обе стороны убедились что другая сторона жива и готова к общению. Без этого можно отправлять данные в пустоту.

**Где используется:** HTTP, HTTPS, SSH, FTP, SMTP — везде где важна целостность данных.

### Что такое UDP

**UDP (User Datagram Protocol)** — протокол передачи данных без установки соединения и без подтверждения доставки. Отправил пакет и забыл.

**Как работает:**
1. Берёт данные, упаковывает в пакет
2. Отправляет адресату
3. Всё. Не проверяет дошёл ли пакет, не пересылает потерянные

**Где используется:** DNS, видео-стриминг, онлайн-игры, VoIP — где важнее скорость чем потеря отдельных пакетов.

### Сравнение

| TCP | UDP |
|-----|-----|
| Надёжный — подтверждает доставку | Быстрый — без подтверждения |
| Устанавливает соединение | Без соединения |
| Пересылает потерянные пакеты | Не пересылает |
| Сохраняет порядок пакетов | Порядок может нарушиться |
| Используется для важных данных | Используется когда скорость важнее |
| HTTP, HTTPS, SSH, FTP | DNS, видео-стриминг, игры |
